什么是SVG?

  • SVG指可伸缩矢量图形(Scalable Vector Graphics)
  • SVG用于定义 使用在网络的基于矢量的图形
  • SVG使用XML格式定义图形
  • SVG图形在放大或改变尺寸的情况下其图形质量不会有损失
  • SVG是W3C的标准

SVG的优势

    与其他图像格式相比(如: JPEG GIF),使用SVG的优势在于:
  • SVG图像可通过文本编辑器来创建和修改
  • SVG图像可被索引 搜索 脚本化 压缩
  • SVG图像可伸缩
  • SVG图像可在任何的分辨率下被高质量的打印
  • SVG可在图像质量不下降的情况下被放大
Internet Explorer 9、Firefox、Opera、Chrome 以及 Safari 支持内联 SVG。

Canvas SVG都允许在浏览器中创建图形,但是他们在根本上是不同的

    SVG

  • SVG是一种使用XML描述2D图形的语言
  • SVG基于XML,这意味这SVG DOM中的每个元素都是可用的.可以为某个元素附加 JavaScript 事件处理器
  • SVG中,每个被绘制的图形均被视为对象.如果SVG对象的属性发生变化,那么浏览器能够自动重现图形

Canvas

  • Canvas 通过JavaScript 来绘制2D图形
  • Canvas是挨个像素进行渲染的
  • 一旦图像被绘制完成, 他就不会继续得到浏览器的关注.如果其位置发生变化,那么整个画布也将重新绘制,包括任何或许已被图形覆盖的对象

两个对比

    Canvas

  • 依赖分辨率
  • 不支持事件处理器
  • 弱的文本渲染能力
  • 能够以.png或.jpg格式保存结果图像
  • 最适合图像密集型的游戏,其中的许多对象会被频繁绘制

SVG

  • 不依赖分辨率
  • 支持事件处理
  • 最适合带有大型渲染区域的应用程序(比如googlemap)
  • 复杂度高会减慢渲染速度(任何过渡使用DOM的应用都不快)
  • 不适合游戏应用
