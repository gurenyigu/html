input元素

<input> 是最重要的 表单元素

input 的形态取决于 type属性

type
值 描述
text 定义常规文本输入.换行符将自动从输入值中删除
submit 定义提交按钮(提交表单)
image 图形提交按钮.必须使用src属性来定义图像的源和alt属性来定义替代文本.可以使用height width来定义图像大小(单位px)
button 没有默认行为的按钮
checkbox 复选框允许选择/取消选择单个值
radio 定义单选按钮输入(选择多个的一个)使用此方法必须给每个设置value值.name设置统一值
file 允许用户选择文件的控件.使用accept属性来定义控件可以选择的文件类型
hidden 不显示, 但其值交给服务器的控件
password 其 值被屏蔽的单行文本输入. 使用maxlength属性来指定可以输入的值的最大长度
reset 将表单内容重置为默认值的按钮
color 用于指定颜色的控件.颜色选择器的UI没有必要的功能
date 用于输入日期的控件(年月日, 没有时间)
datetime-local 用于输入日期和时间的控件, 没有时区
email 用于编辑电子邮件地址的字段
month 用于输入一个月和一年的控件,没有时区
number 用于输入数字的控件
range 用于输入确切值不重要的数值的控件
search 用于输入搜索字符串的单行文本字段.换行符将自动从输入值中删除
tel 用于输入电话号码的控件.换行符将自动从输入值中删除,但不会执行其他语法.您可以使用诸如 psttern和maxlength之类的属性来限制在控件中输入的值.在:valid 与 :invalid CSS伪类中酌情应用
time 用于输入没有失去的时间值的控件
url 用于编辑URL的字段.在提交之前,输入值被验证为包含空字符串或有效的绝对URL.
week 用于输入不含时区的星期数和周号的日期控件

注：HTML表单验证是不为脚本，确保输入的数据是正确的格式的替代品。有人对HTML进行调整太容易了，允许他们绕过验证，或者完全删除它。某些人也可能完全绕过您的HTML，并将数据直接提交到您的服务器。如果您的服务器端代码无法验证其接收到的数据，则在数据库中输入格式不正确的数据（或数据太大，类型错误等）的数据库时，灾难可能会发生。

<input>全局属性
属性 值/描述
type 要呈现的控件的类型.
name 控件名称，他与表单一起提交
accept 如果type类型为file,则该属性将指示服务器接受的文件类型,否则将被忽略.该值必须是以逗号分隔的唯一内容类型说明符类表:
  • 以STOP字符(U+002E)开始的文件拓展名(例如: ".jpg,.png,.doc")
  • 一个有效的MIME类型,但没有拓展名
  • audio/*表示音频文件
  • video/*表示视频文件
  • image/*表示图片文件
checked 如果该元素的 type属性的值为 radio 或者 checkbox,则该布尔属性的存在制定了该空间的默认选择状态
disabled 这个布尔属性表示此表单控件不可用.特别是在禁用的控件中,click事件(不会触发).并且,禁用的控件的值在提交表单时也不会被提交.当type值为hidden,此属性别忽略
src 如果type属性的值为image，则该属性指定图像的位置的URI,以在image提交按钮上显示，否则将忽略.
value 控制初始值.该属性是可选的，除非type属性的值是　radio checkbox．请注意，当重新加载页面时，如果在重新加载之前更改了值，Gecko和IE将忽略HTML源代码中指定的值．
autocomplete 这个属性表示这个控件的值是否可被浏览器自动填充.如果type属性的值是 hidden checkbox radio file.或为按钮类型(botton submit reset image)则本属性被忽略.可用值:
  • off: 用户必须手动填充,或者该页面提供了自己的自动补全方法.浏览器不对此字段进行自动填充.
  • on: 浏览器可以根据用户先前的填表情况对此字段自动填充
  • name: 完整的姓名
  • 更多参考: 火狐开发者网站
  • 如果 input 元素上没有autocomplete属性,浏览器可以取包含其的 form元素的autocomplete属性
autofoucus 这个布尔值的属性允许指定表单控件则页面加载时具有焦点(自动获得焦点),除非用户将其覆盖,例如通过键入不同的控件.文档中只有一个表单元素可以具有autofocus属性.如果type属性设置为隐藏则不能应用
autosave 该属性应定义为唯一值.如果type属性的值是搜索,则以前的搜索项值将保持在页面加载的下拉列表中.
form input元素与其关联的 form 元素(其表单所有者). 属性的值必须是同一文档中的 form元素的ID.如果未指定此属性,则此input 元素必须是form 元素的后代. 此属性使 input 元素可以在文档中任意位置,而不仅仅作为 form 的后代元素
formaction 处理由 input元素提交的信息(如果他是 submit button image)的程序的URI.如果指定,他将覆盖form 元素有所者的 action 属性
formenctype 如果 input元素是submit button image, 则此属性指定用于将表单提交到服务器的内容的类型.可能的值是:
  • application/x-www-form-urlencoded: 如果未指定属性则为默认属性.
  • multipart/form-data: 如果使用type属性设置为file的 input 元素,请使用此值.
  • text/plain
如果指定了这个属性,他将覆盖元素表单所有者的 enctype 属性.
formmethod 如果 input 元素是 submit button image,则此属性指定浏览器用于提交表单的HTTP方法.可能值为:
  • post: 表单中的数据包含在表单的正文中,并发送到服务器
  • get: 表单中的数据附加到表单属性URI中,带有 "?" 作为分隔符,并将生成的URI发送到服务器.当表单没有side-effects并且只包含ASCII字符时,请使用这个方法.
如果指定,此属性将覆盖元素表单所有者的method属性.
formnovalidate 如果 input 元素是 button submit image, 则此布尔属性指定表单在提交时不被验证.如果指定了此属性,他将覆盖元素表单所有者的novalidate属性
formtarget 如果 input 元素是 submit button image,则此属性是一个名称或关键字,指示在提交表单后收到的响应的显示位置. 这是浏览上下文的名称或关键字(例如,选项卡,窗口或内联框架).如果指定了此属性,他将覆盖元素表单所有者的目标属性.
  • _self: 将响应加载到与当前浏览环境相同的浏览上下文中
  • _blank: 将响应加载到新的未命名浏览上下文中
  • _parent: 将响应加载到当前父浏览上下文中.如果没有父级,此选项的行为方式与_self相同
  • _top: 将响应加载到顶级浏览上下文中(即,作为当前浏览上下文的浏览上下文,并且没有父级)如果没有父级,此选项的行为方式与_self相同
width 如果type属性的值为　image,则此属性定义为按钮显示的图像的宽度.
height 如果type属性的值为 image, 则此属性定义为按钮显示的图像的高度
list 标识给用户建议的预定义选项列表. 该value必须是同一文档中的 <datalist> 元素的　id. 浏览器只显示这个 input 元素的有效值的选项.当　type 的属性值为　hidden checkbox radio file button 时，忽略此属性，
max 这项最大(数字或日期时间)值，不得小于其最小(min属性)值
min 这项最小(数字或日期时间)值，不得大于其最大值(max属性)值
step 使用最小和最大属性来限制可以设置数字或日期时间值的增量．　它可以是任何字符串或正的浮点数．如果此属性未设置值．则控件仅接受大于最小值的步长值的倍数的值．
maxlength 如果type的值是　text email search password tel url，　那么这个属性指明了用户最多可以输入的字符个数(按照Unicode编码方式计数)．对于其他类型的输入框，该属性被忽略．它可以大于　size属性的值．
如果不指定这个属性，那么用户可以输入任意多的字符(默认行为)．
如果指定为一个负值，那么元素表现出默认行为.
这个属性的约束在value值发生变化时执行． IE10+
multiple 这个布尔属性值　指定用户是否可以输入多个值．当属性设置为　email file时，此属性适用．其他属性则被忽略
pattern 检查控件值的正则表达式．pattern必须匹配整个值，而不仅仅是某些子集．使用　title属性来描述帮助用户的模式．当type属性的值为 text search tel url email时，此属性使用，否则将被忽略. IE10+
placeholder 提示用户可以在控件中输入什么．占位符文本中不可以包含　回车　换行．该属性适用于type属性为 text search tel url email中，否则非忽略．
readonly 此布尔属性值表示用户无法修改控件的值．如果type属性为 hidden range color checkbox radio file button类型，则忽略次属性.
required 这个属性指定用户在提交表单之前必须为该元素填充值，当type属性是: hidden image button(submit reset button)时不可用．:optional 和　:required　CSS伪元素的样式将可以对该字段应用更改外观
selectionDirection 选择发生的方向．如果在ＬＴＲ区域设置中从左到右进行选择(正向选择)，则值为"forward", 反向选择则为"backward",如果选择方向未知，则可以为"none"
size 控件的初始大小.值为像素，除非type属性的值为 text password，在这种情况下，该值为整数个字符．从HTML5开始，此属性仅在type属性设置为　text search tel url email password时适用．此外，size必须大于 0.如果不指定大小　，则默认为２０
spellcheck 将此属性的值设置为true表示元素需要检查其拼写和语法．值default表示该元素即将根据默认行为进行操作，可能基于父元素自己的拼写检查．置为false表示不应该检查元素.
