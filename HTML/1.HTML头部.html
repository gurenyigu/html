HTML <head> 元素

head 元素是所有头部元素的容器. 其中可以添加的标签为: <title> <base> <link> <meta> <script> <style> 下面一一介绍

<title>元素

用于定义文档标题, 必须属性

  • 定义浏览器工具栏中的标题
  • 提供页面被添加到收藏夹时显示的标题
  • 显示在搜索引擎结果中的页面标题
属性 值 描述
dir rtl ltr 规定元素中内容的方向
lang language_code 规定元素中内容的语言代码
xml:lang language_code 规定XHTML文档中元素内容的语言代码

<base>

此标签为页面上的所有链接规定默认地址或默认目标(target)属性

        <base target="_blank">
属性 值 描述
href URL 规定页面中所有相对链接的基准URL. 在通常情况下,浏览器会从当前文档的URL中提取响应的元素来填写相对URL中的空白. 使用 base 标签可以改变这一点. 浏览器所有将不再使用到当前文档URL, 而使用指定的基本URL 来解析所有的相对URL. 这其中收益的包括: a img link form 标签中的URL.
target

<link>

link 标签定义文档与外部资源之间的关系

HTML中 link标签没有结束标签

在XHTML中, link标签必须正确的关闭

link中只包含属性, 此元素只能存在 head中, 不过它可以存在任意次数

        <link rel="stylesheet" type="text/javascript" href="jquery-min.js" >
        <link rel="stylesheet" type="text/css" href="style.css">
属性 值 描述
href URL 规定被链接文档
rel alternate author help icon licence next pingback prefetch prev search sidebar stylecheet tag 规定当前文档与被链接文档之间的关系
sizes heightxwidth | any 规定被链接资源的尺寸, 适用于 rel="icon"
  • heightxwidth: 为链接图标会定一个或多个以像素单位的宽高值.由x 或 X间隔
  • any: 规定图标是可伸缩的(比如 SVG 图标)
type MIME_type 规定被链接文档的 MIME类型
media media_query 规定被链接文档将被显示在什么设备上(所有浏览器支持: screen print all)

<style> 元素

此标签用于为HTML文档定义样式

        <style type="text/css"> 
          a {
              color="red";
          }
        </style>
  • 此标签用于为HTML文档定义样式信息.
  • type 属性时必须的, 定义style元素内容. 唯一可能值为 text/css
  • 位于 head 部分中
  • 其还存在一个可选属性 media. 同link标签的一样

<meta> 元素

元shug(metadata)是关于数据的信息

meta 标签提供关于HTML 文档的元数据. 元数据不会显示在页面上, 但是对于机器是可读的.

典型的情况是, meta元素被用于规定页面的描述 关键词 文档的作者 最后修改时间及其它元数据

meta始终位于 head 元素中. 元数据可用于浏览器(如何显示内容或重新加载页面), 搜索引擎(关键字),或其他WEB服务

针对搜索引擎关键字

一些搜索引擎会利用meta元素的name 和 content 属性来索引当前页面: 下面进行页面描述

        <meta name="description" content="复习 HTML, CSS, JAVASCRIPT" />

下面的meta元素定义页面的关键词

      <meta name="keywords" content="HTML, CSS, XML" />
属性 值 描述
content(必须属性) some_text 定义与 http-equiv 或 name属性相关的元信息
  • content属性提供了名称/值对中的值. 该值可以是任何有效的字符串
  • content属性始终要和name属性或http-equiv属性一起使用
http-equiv content-type expires(到期) refresh(刷新) set-cookie 把content属性关联到HTTP头部
  • http-equiv 属性为名称/值对提供了名称. 并指示服务器在发送实际的文档前先在要传送给浏览器的 MIME 文档头部包含名称/值对
  • 当服务器向浏览器发送文档时,会先发送许多名称/值对.虽然有些服务器会发送许多这种名称/值对,但是所有服务器都至少要发送一个: content-type:text/html. 这会告诉浏览器准备接收一个HTML文档
  • 使用带有 http-equiv 属性的 <meta> 标签时,服务器将把名称/值对添加到发送给浏览器的内容头部. 例如添加: 
                                <meta http-equiv="charset" content="iso-8859-1">
                            <meta http-equiv="expires" content="31 Dec 2018">

    这样的话 发送到浏览器的头部就应该包含:

                                content-type: text/html
                            charset: iso-8859-1
                            expires: 31 Dec 2018

    只有浏览器可以接收这些头部字段, 并能以适当的方式使用他们时,这些字段才有意义

                                    例子:(如果网站更改了URI, 那么可以使用 meta 进行重定向)
                                <meta http-equiv="Content-Type" content="text/html; charset=gb2312" />
                                <meta http-equiv="Refresh" content="5; url=https://www.baidu.com" />
name author(作者) description(描述) keywords generator(引擎) revised(修订) others 把　content 属性关联到一个名称
  • name属性提供了名称/值对中的名称. HTML 和 XHTML 标签都没有指定任何预先定义的 meta 名称. 通常情况下, 您可以自由使用对自己和源文档的读者来说有意义的名称.
  • "keywords"是一个经常被用到的名称. 他为文档定义了一组关键字.某些搜索引擎在遇到这些关键字时,会用这些关键字对文档进行分类
  • 类似这样该的meta标签可能对与进入搜索引擎有所帮助: 
                                <meta name="keywords" content="HTML,ASP,PHP,SQL">
  • 如果没有提供 name 属性, 那么名称/值对中的名称会采用 http-equiv属性的值
scheme some_text 定义用于翻译content属性的格式. 应该在由 head 标签的 profile 属性指定的概况文件进行了定义

<script> 元素

此标签用于定义客户端脚本, 如:javascript

script 元素既可以包含脚本语句, 也可以通过 src 属性指向外部脚本文件

必须的type属性规定脚本的MIME类型

JavaScript的常见应用 图像操作 表单验证 动态内容更新

如果在frameset 标签后,将被忽略;元素中的代码没有位于哪个函中,那么那些代码会在页面被加载时立即执行.

元素中的代码没有位于哪个函中,那么那些代码会在页面被加载时立即调用.

XHTML 与 HTML4 使用略有不同, 在XHTML中:

        <script type="text/javascript"><![CDATA[
            document.write("Hello World")
         //]]></script>
属性 值 描述
async (H5) async
  • async 属性规定一旦脚本可用, 则会异步执行
  • async 属性仅适用与外部脚本(只有在使用SRC属性时)
    1. 如果async="async" 脚本相对于 页面的其余部分异步的执行(当页面继续进行解析时,脚本也将执行)
    2. 如果不使用async且defer="defer": 脚本将在页面完成解析时执行
    3. 如果即不使用async 也不使用 defer: 在浏览其继续解析前, 立即读取并执行脚本
defer (仅在IE中可用) defer defer属性规定是否对脚本执行进行延迟, 直到页面加载完毕
charset charset(UTF-8)
  • charset属性规定在外部脚本文件中使用的字符编码
  • 如果外部文件中的字符编码与主文件中的编码方式不同,就要用到charset属性
  • 默认字符编码为: ISO-8859-1

src 和 charset 属性

对于特别长的 JavaScript 程序或者经常重复使用的教程来说,你可以希望这些代码存放到一个单独的文件中. 这种情况下,让浏览器通过 src 属性来载入那个单独的文件

src 的值是包含这个JavaScript程序的文件的URL. 保存的文件的MIME类型应是 application/x-javascript, 但如果文件名的后缀为.js, 也能够被正确配置了的服务器进行恰当的处理

                        
指定下载名为 jquery.js 的 JavaScript程序

                        <script type="text/javascript" src="jqurey.js" > </script>

charset 属性与 src 属性一起使用, 告诉浏览器用来编码这个javascript程序的字符集.

Unicode标准涵盖了世界上所有的字符,表点和符号,不伦使用何种平台 程序 语言 Unicode都能够进行文本数据的处理 存储 交换.

UTF-8, 它的字符可以是1-4个字节长, UTF-8卡一表示Unicode标准中的任意字符. UTF-8向后兼容ASCII. 他是网页 电子邮件的首选编码
src URL 外部脚本的URL,可能值:
  • 绝对URL, 指向其他站点
  • 相对URL, 指向本站内文件
